So wird der Radar «echt»

Der Radar zeigt echte Zürcher Baugesuche, sobald eine schlanke Karte-Datei baugesuche.geojson neben dieser Seite liegt. Diese wird aus den offiziellen, täglich aktualisierten Kantonsdaten erzeugt:
  1. Amtliche Quelle (täglich): daten.statistik.zh.ch/ogd/daten/ressourcen/KTZH_00002982_00006403.gpkg
  2. Das mitgelieferte Skript update-baugesuche.py lädt diese Datei, rechnet die Koordinaten von LV95 nach WGS84 um und schreibt baugesuche.geojson.
  3. Für den Automatik-Betrieb (täglicher Scan) läuft dieses Skript einmal pro Tag – lokal per Aufgabenplanung oder gratis via GitHub Action.

Ehrliche Grenzen

Der offene Datensatz enthält Projektstandort, Kataster-/BFS-Nummer und Projektbeschrieb, aber keine Namen/Adressen der Bauherrschaft (Datenschutz). Abdeckung: nur Kanton Zürich. Ein gefiltertes Baugesuch belegt kein realisiertes Bauvorhaben.

E-Mail-Versand ist hier als Vorschau simuliert – der produktive Versand läuft über den täglichen Hintergrund-Job. Es werden keine Daten fabriziert: ohne verbundene Quelle zeigt der Radar keine Baugesuche. Projekte werden lokal im Browser gespeichert.
